# Codex 红绿灯设计说明

## 目标

做一个纯本地 Windows EXE，用悬浮红绿灯显示 Codex CLI 当前运行状态。程序不需要服务器、不需要云服务，也不依赖在线更新接口。

## 目标平台

- 只支持 Windows。
- 使用 C# WPF 原生桌面程序。
- 最终打包成可直接运行的 `.exe`。
- 所有状态、配置、统计数据默认保存在 `%USERPROFILE%\.codex` 目录下。如果环境变量 `CODEX_HOME` 存在，则优先使用 `CODEX_HOME` 指向的目录。

## 用户体验

程序启动后，在主屏幕右上角显示一个始终置顶的小窗口。默认样式是竖向三灯红绿灯，外壳为深色圆角样式。当前状态对应的灯会发光，并带轻微呼吸或闪烁效果。

状态定义：

- 红灯：Codex 已收到用户任务，正在处理。
- 黄灯：Codex 正在等待权限确认。第一版不承诺覆盖所有普通“等待用户回复”的情况，因为 Codex 当前可稳定挂钩的是 `PermissionRequest`。
- 绿灯：Codex 当前轮次已结束，或者刚启动会话。
- 灰灯或暗灯：状态文件缺失、损坏，或者状态值无法识别。不要仅凭更新时间过旧就切灰灯，避免 Codex 长时间处理任务时被误判为空闲。

窗口可以拖动。程序会保存上次窗口位置，下次启动时恢复。窗口右下角有齿轮按钮，点击后打开一个小设置面板，包含静音和深浅色模式开关。

程序必须做单实例限制，避免同时启动多个 EXE 导致多个托盘图标、重复窗口和重复统计。

## 托盘菜单

系统托盘图标右键菜单包含：

- 切换到红灯。
- 切换到黄灯。
- 切换到绿灯。
- 切换到单灯样式或三灯样式。
- 查看配置路径。
- 重新写入配置。
- 切换浅色模式或深色模式。
- 本周周报。
- 检查更新。
- 关于我。
- 退出。

因为这个版本是纯本地版本，`检查更新` 不访问网络，只提示“本地版不使用在线更新检查”。

## Codex 集成方式

集成方式使用 Codex 生命周期 hooks。程序只安装和更新自己管理的 hooks，不覆盖用户已有 hooks。

hooks 配置文件：

```text
%CODEX_HOME%\hooks.json 或 %USERPROFILE%\.codex\hooks.json
```

hook 辅助脚本：

```text
%CODEX_HOME%\codex-traffic-light\codex_traffic_light_write_status.ps1 或 %USERPROFILE%\.codex\codex-traffic-light\codex_traffic_light_write_status.ps1
```

状态文件：

```text
%CODEX_HOME%\codex_traffic_light_state.json 或 %USERPROFILE%\.codex\codex_traffic_light_state.json
```

统计文件：

```text
%CODEX_HOME%\codex_traffic_light_stats.json 或 %USERPROFILE%\.codex\codex_traffic_light_stats.json
```

程序管理的 hooks：

- `UserPromptSubmit`：写入红灯状态。
- `PermissionRequest`：写入黄灯状态。
- `Stop`：写入绿灯状态。
- `SessionStart`：写入绿灯状态。

写入 hooks 后，程序必须提示用户在 Codex 中运行 `/hooks` 并信任这些 hook。Codex 对未信任的非托管命令 hook 会直接跳过。

hooks 不直接内嵌复杂 PowerShell JSON 拼接命令，而是调用本地辅助脚本。这样可以降低 Windows 命令行引号、反斜杠、用户目录空格导致的失败概率。

## 状态文件格式

```json
{
  "state": "red",
  "event": "UserPromptSubmit",
  "updatedAt": "2026-06-01T10:45:00+08:00"
}
```

允许的 `state` 值：

- `red`
- `yellow`
- `green`
- `unknown`

## 架构

使用一个小型 .NET 解决方案，拆成三个项目：

- `CodexTrafficLight.Core`：负责本地路径、状态模型、状态读写、统计、hooks 安装、设置持久化。
- `CodexTrafficLight.App`：负责 WPF 界面、托盘菜单、动画、文件监听、用户提示。
- `CodexTrafficLight.Tests`：负责核心逻辑的单元测试。

这样可以在不启动 WPF 窗口的情况下测试 Codex 文件处理逻辑。

## 验证标准

第一版可用前至少要验证：

- Core 单元测试通过。
- 托盘菜单手动切换红黄绿时，界面状态正确变化。
- hook 安装器能合并已有 `hooks.json`，不会删除无关 hooks。
- 状态文件变化后，WPF 灯色能自动更新。
- 发布后的 EXE 可以启动，并能重新写入 hooks。
- 重复启动 EXE 时，只保留一个运行实例。

## 第一版不做的内容

- 在线更新。
- 服务器端统计。
- 跨平台支持。
- MSI 安装包。
- 自动绕过 hook 信任机制。
- 深度读取 Codex 内部 SQLite 数据库或 transcript 文件。
